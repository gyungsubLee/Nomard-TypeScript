# ✍️ 객체 지향 프로그래밍(Object-Oriented Programming(OOP))
초기 절차적 프로그래밍을 보안하기 위해 나온 패러다임으로 큰 문제를 작게 쪼개는 것이 아닌, 
먼저 **작은 문제들을 해결해 나갈 수 있는 객체들을 만든 뒤, 이 객체들을 조합해 큰 문제를 해결**하는 상향식(Bottom-up)해결 방법이다. 따라서 프로그램 **구현에 필요한 객체를 파악**하고 **각각의 객체들의 역활이 무엇인지 정의**하여 **객체들 간의 상호작용**을 통해 프로그램을 설계한다.
<br/>

객체 지향 프로래밍(OOP)은 실제 세계를 **객체**로 모델링해 **데이터(property)** 와 **절차(method)** 로 묶어서 구현한다.

<br/><br/>

## ✍️ 객체 지향 프로그래밍의 특징
### 1. 추상화(abstraction)
**객체들의 공통적인 특징(기능, 속성)을 도출하는 것**을 의미한다.<br>
-> **class를 통해 instance(구현된 객체(실체))를 정의**하기 때문에 **클래스를 정의**하는 것을 **추상화**라고 할 수 있다.

<br/>

### 2. 캡슐화(encapsulation)
객체가 독립적으로 역활을 할 수 있도록 **데이터(변수)와 기능(함수)을 하나로 묶는 것**을 의미한다. 즉, 데이터의 번들링(bundling, 묶다)이다.
이 번들링은 클래스를 통해 구현되고, 해당 클래스의 인스턴스 생성을 통해 클래스 안에 포함된 멤버 변수와 메소드에 쉽게 접근할 수 있다.

- **은닉화(information hiding)** <br>캡슐화로부터 파생된 보조 개념으로 
**실제로 구현되는 부분**을 외부에 드러나지 않도록 특정 모듈 내부로 **감추는(은닉하는) 것**이다.<br><br>
내부 구현은 감추고 모듈 내에서의 응집도를 높이며, 외부로의 노출을 최소화하여 모듈 간의 결합도를 떨어뜨려 유연함과 유지보수성을 높여준다. <br><br>
일반적으로 3종류의 접근 접근 제한(**public, protected, private**)이 사용된다. 
(자세한 내용은 위의 field 정리 부분을 참조) 

<br/>

### 3. 상속(inheritance)
**하나의 클래스(부모 클래스)가 가진 특징(함수, 데이터)** 을 **다른 클래스(자식 클래스)가 그대로 물려받는 것** 이다. <br>
상속된 객체 기능(함수, 데이터)는 확장 및 변경 가능하다. (오버라이딩(overriding)) - OCP(Open-Closed-Principle) 
<br>
 코드(클래스) 재사용성 증가 시켜준다.

<br/>

### 4. 다형성(polymorphism)
하나의 변수, 또는 함수가 상황에 따라 다른 의미로 해석될 수 있을 말한다.
<br>
위 TypeScript에서는 제네릭(<>)을 통해 구현된다.

<br><br>

## ✍️ 객체 지향 5원칙(SOLID)
객체지향에서 꼭 지켜야 할 5개의 원칙

### 1. SRP(Single Responsibility Principle): 단일 책임 원칙

<br>

### 2. OCP(Open-Closed-Principle): 개방-폐쇠 원칙
객체는 **확장은 개방적**, **수정은 폐쇠적**이어야 한다는 원칙이다.
즉, **객체 기능 확장을 허용하고 스스로의 변경은 피해야한다.**

<br>

### 3. LSP(Liskov Substitution Principle):리스코프 치환 원칙

<br>

### 4. ISP(Interface Segregation Principle): 인터페이스 분리 원칙

<br/>

### 5. DIP(Dependency Inversion Principle): 의존성 역전 원칙
